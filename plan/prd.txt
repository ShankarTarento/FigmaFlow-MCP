4. Functional Requirements
4.1 Figma Design Input

Accept Figma file URL

Accept optional node ID

Validate access permissions

Fetch design data using Figma API

Acceptance Criteria

User can paste a Figma URL

System fetches design metadata successfully

Errors are shown clearly if access fails

4.2 MCP Orchestration Layer

MCP server runs independently (local/cloud)

Exposes tool-based APIs:

get_figma_design

generate_flutter_widget

generate_widget_tests

generate_qa_testcases

Handles AI model invocation

Produces structured output

Acceptance Criteria

Tools are callable via HTTP

Inputs/outputs follow defined schemas

Server is stateless for MVP

4.3 Code & Test Generation
Flutter Widget Generation

Generate valid Flutter widget code

Respect layout hierarchy

Use Material widgets by default

Widget Test Generation

Generate Flutter widget tests

Cover basic rendering and interactions

Use best practices (pumpWidget, finders)

QA Test Case Generation

Generate readable test cases

Output in text or JSON

Cover UI states and validations

Acceptance Criteria

Generated code compiles

Tests run successfully with minimal edits

Output is deterministic and readable

Generated output must be Copilot-friendly
